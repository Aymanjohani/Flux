# Communication Guidelines

## Newsletter Standards

### Tone & Style
- **Professional, corporate tone** - NOT casual or conversational
- **NO emojis** - Clean, professional text only
- **Formal language** - Business communication standard
- **Clear structure** - Headers, sections, bullet points
- **Concise** - Respect reader's time

### Template Requirements
- HTML format with proper structure
- Professional color scheme (blues, grays, whites)
- Clean typography
- Mobile-responsive design
- Company branding where appropriate

### Content Structure
1. **Executive Summary** - Key highlights at top
2. **Main Sections** - Clear headers for each topic
3. **Metrics/Data** - Tables or lists, not prose
4. **Action Items** - Clear next steps if applicable
5. **Footer** - Professional sign-off

## Internal vs External Communication

### Internal (Team Chat, Telegram)
- Can be more casual
- Emojis acceptable
- Quick updates fine
- Conversational tone okay

### External (Emails, Newsletters, Reports)
- Always professional
- No emojis
- Formal structure
- Corporate voice

## Email Formatting

### To/CC Rules
- Verify email addresses before sending
- Cross-reference with team.md for correct addresses
- Don't assume email format (check actual addresses)
- CC: Use sparingly, only when truly needed

## Meeting Communications

### Calendar Invites
- Clear, descriptive subject
- Full context in description
- Correct attendee list (verify against project team)
- Include meeting link
- Set appropriate reminder

### Meeting Summaries
- Send within 24 hours
- Clear action items
- Decisions documented
- Next steps identified
- All attendees CC'd

## Validation Checklist

Before sending any formal communication:
- [ ] Tone matches context (professional for external)
- [ ] No emojis in professional communications
- [ ] Email addresses verified against team.md
- [ ] Recipients appropriate for context
- [ ] Clear subject/purpose
- [ ] Proofread for errors
- [ ] Action items clearly stated
